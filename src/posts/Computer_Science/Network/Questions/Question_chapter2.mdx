### Section 2.1: Fundamental Concepts


1. List five nonproprietary Internet applications and the application-layer protocols that they use.<br/>
	다섯 가지 비소유 인터넷 애플리케이션과 그들이 사용하는 응용 계층 프로토콜을 나열하십시오.
	<Answer>
		**웹 브라우징** - 사용 프로토콜: HTTP (Hypertext Transfer Protocol) 및 HTTPS (HTTP Secure)<br/>
		**이메일** - 사용 프로토콜: SMTP (Simple Mail Transfer Protocol), POP3 (Post Office Protocol 3), IMAP (Internet Message Access Protocol)<br/>
		**파일 전송** - 사용 프로토콜: FTP (File Transfer Protocol) 및 SFTP (SSH File Transfer Protocol)<br/>
		**도메인 이름 시스템 (DNS)** - 사용 프로토콜: DNS (Domain Name System)<br/>
		**인스턴트 메시징 및 채팅** - 사용 프로토콜: XMPP (Extensible Messaging and Presence Protocol)
	</Answer>

2. What is the difference between network architecture and application architecture?<br/>
	네트워크 아키텍처와 응용 프로그램 아키텍처의 차이점은 무엇입니까?
	<Answer>
		**네트워크 아키텍처**는 네트워크의 구조와 구성 요소를 설명하는 것입니다.<br/> 
		**응용 프로그램 아키텍처**는 응용 프로그램의 구조와 구성 요소를 설명하는 것입니다.
	</Answer>

3. For a communication session between a pair of processes, which process is the client and which is the server?<br/>
	한 쌍의 프로세스 간의 통신 세션에서 어떤 프로세스가 클라이언트이고 어떤 프로세스가 서버입니까?
	<Answer>
		클라이언트 프로세스는 통신 세션을 시작하는 프로세스이고 서버 프로세스는 클라이언트 프로세스의 요청에 응답하는 프로세스입니다.
	</Answer>

4. Do you agree with the statement, “There is no notion of client and server sides of a communication session” in a P2P file-sharing application? Why or why not?<br/>
	P2P 파일 공유 응용 프로그램에서 "통신 세션의 클라이언트 및 서버 측 개념이 없다"는 주장에 동의하십니까? 그 이유는 무엇입니까?
	<Answer>
		동의하지 않습니다. P2P 파일 공유 응용 프로그램에서도 클라이언트와 서버의 개념이 존재합니다. P2P 응용 프로그램에서도 클라이언트와 서버의 역할이 존재하며, 클라이언트는 파일을 요청하고 서버는 파일을 제공하는 역할을 합니다.
	</Answer>

5. What information is used by a process running on one host to identify a process running on another host?<br/>
	한 호스트에서 실행 중인 프로세스가 다른 호스트에서 실행 중인 프로세스를 식별하는 데 사용하는 정보는 무엇입니까?
	<Answer>
		프로세스가 다른 호스트에서 실행 중인 프로세스를 식별하는 데 사용하는 정보는 **IP 주소**와 **포트 번호**입니다.
	</Answer>

6. Suppose you wanted to do a transaction from a remote client to a server as fast as possible. Would you use UDP or TCP? Why?<br/>
	가능한 한 빠르게 원격 클라이언트에서 서버로 트랜잭션을 수행하려면 UDP 또는 TCP를 사용하겠습니까? 그 이유는 무엇입니까?
	<Answer>
		UDP를 사용할 것입니다. UDP는 TCP보다 더 빠르게 데이터를 전송할 수 있습니다. UDP는 데이터 전송을 보장하지 않지만, 데이터 전송 속도가 빠르기 때문에 트랜잭션을 빠르게 수행할 수 있습니다.
	</Answer>

7. Referring to Figure 2.4, can you conceive of an application that requires no data loss and that is also highly time-sensitive?<br/>
	그림 2.4를 참조하여 데이터 손실이 없고 시간에 민감한 응용 프로그램을 상상할 수 있습니까?
	<Answer>
		데이터 손실이 없고 시간에 민감한 응용 프로그램의 예로는 **실시간 스트리밍 서비스**가 있습니다. 실시간 스트리밍 서비스는 데이터 손실이 없어야 하며, 데이터 전송이 빠르게 이루어져야 합니다.
	</Answer>

8.  List the four broad classes of services that a transport protocol can provide. For each of the service classes, indicate if either UDP or TCP (or both) provides such a service.<br/>
	전송 프로토콜이 제공할 수 있는 네 가지 넓은 범주의 서비스를 나열하십시오. 각 서비스 클래스에 대해 UDP 또는 TCP (또는 둘 다)가 그러한 서비스를 제공하는지 여부를 나타내십시오.
	<Answer>
		1.{} **Connection-oriented service** - TCP<br/>
		2.{} **Reliable data transfer** - TCP<br/>
		3.{} **Throughput** - UDP<br/>
		4.{} **Timing** - UDP
	</Answer>

9. Does TLS operate at the transport layer or the application layer? If the application developer wants TCP to be enhanced with TLS, what does the developer have to do?<br/>
	TLS는 전송 계층에서 작동합니까 아니면 응용 계층에서 작동합니까? 응용 프로그램 개발자가 TCP를 TLS로 향상시키길 원한다면, 개발자는 무엇을 해야합니까?
	<Answer>
		TLS는 **응용 계층**에서 작동합니다. 응용 프로그램 개발자가 TCP를 TLS로 향상시키길 원한다면, 개발자는 **TLS 라이브러리**를 사용하여 TLS를 구현해야 합니다.
	</Answer>

### Section 2.7: Socket programming with UDP and TCP

26. In Section 2.7, the UDP server described needed only one socket, whereas the TCP server needed two sockets. Why? If the TCP server were to support n simultaneous connections, each from a different client host, how many sockets would the TCP server need?<br/>
	섹션 2.7에서 설명된 UDP 서버는 하나의 소켓만 필요로 했지만, TCP 서버는 두 개의 소켓이 필요했습니다. 왜 그런가요? TCP 서버가 서로 다른 클라이언트 호스트에서 n개의 동시 연결을 지원한다면, TCP 서버는 몇 개의 소켓이 필요합니까?
	<Answer>
		TCP 서버는 **리스닝 소켓**과 **연결 소켓**이 필요합니다. TCP 서버가 서로 다른 클라이언트 호스트에서 n개의 동시 연결을 지원한다면, TCP 서버는 **n+1**개의 소켓이 필요합니다.
	</Answer>

27. For the client-server application over TCP described in Section 2.7, why must the server program be executed before the client program? For the client-server application over UDP, why may the client program be executed before the server program?<br/>
	섹션 2.7에서 설명된 TCP를 통한 클라이언트-서버 응용 프로그램의 경우, 왜 서버 프로그램이 클라이언트 프로그램보다 먼저 실행되어야 하는가요? UDP를 통한 클라이언트-서버 응용 프로그램의 경우, 왜 클라이언트 프로그램이 서버 프로그램보다 먼저 실행될 수 있나요?
	<Answer>
		TCP를 통한 클라이언트-서버 응용 프로그램의 경우, 서버 프로그램이 클라이언트 프로그램보다 먼저 실행되어야 하는 이유는 **서버 프로그램이 클라이언트의 연결 요청을 수락해야 하기 때문**입니다. UDP를 통한 클라이언트-서버 응용 프로그램의 경우, 클라이언트 프로그램이 서버 프로그램보다 먼저 실행될 수 있는 이유는 **UDP가 연결을 설정하지 않기 때문**입니다.
	</Answer>
