### Section 1.2 network edge - end systems, access network, links
4. List four access technologies. Classify each one as home access, enterprise access, or wide-area wireless access.<br/>
	네 가지 접근 기술을 나열하고 각각을 가정용 접근, 기업용 접근 또는 광역 무선 접근으로 분류하세요.
	<Answer>
		**디지털 가입자 회선(DSL)** - **가정용 접근**: DSL은 전통적인 구리 전화선을 통해 데이터를 전송하는 인터넷 연결 유형입니다. 일반적으로 가정용 인터넷 접근에 사용됩니다.<br/><br/>
		**이더넷** - **기업용 접근**: 이더넷은 비즈니스 환경에서 장치를 로컬 네트워크(LAN)에 연결하는 데 주로 사용되는 네트워크 기술입니다. 높은 속도와 신뢰성을 제공합니다.<br/><br/>
		**광섬유** - **기업용 접근/가정용 접근**: 광섬유 기술은 빛을 사용하여 매우 높은 속도로 데이터를 전송합니다. 높은 대역폭이 필요한 기업 환경과 높은 속도의 가정용 인터넷 접근에 모두 사용됩니다.<br/><br/>
		**LTE (장기 진화)** - **광역 무선 접근**: LTE는 모바일 기기 및 데이터 단말기를 위한 무선 광대역 통신의 표준입니다. 넓은 지리적 지역에 걸쳐 고속 데이터 접근을 제공하며 모바일 전화와 데이터 단말기에서 널리 사용됩니다.
	</Answer>

5. Is HFC transmission rate dedicated or shared among users? Are collisions possible in a downstream HFC channel? Why or why not?<br/>
	HFC 전송 속도는 사용자 간에 전용인가요, 아니면 공유인가요? HFC 하류 채널에서 충돌이 가능한가요? 그 이유는 무엇인가요?
	<Answer>
		**HFC(하이브리드 광섬유-동축)** 전송 속도는 사용자 간에 공유됩니다, 전용이 아닙니다. 이 기술은 광섬유와 동축 케이블을 결합하여 광대역 접근 네트워크를 형성하며, 대역폭은 같은 로컬 라인에 연결된 여러 사용자 간에 할당됩니다. 전송 용량이 공유되므로 사용자는 많은 사용자가 동시에 온라인 상태일 때 피크 사용 시간 동안 속도 변동을 경험할 수 있습니다.<br/><br/>
		**HFC 네트워크에서 하류 채널의 충돌은 일반적으로 발생하지 않습니다.** HFC 네트워크에서 하류 채널(제공자에서 사용자로)은 일반적으로 방송 성격을 띱니다. 이는 신호가 한 지점(헤드엔드)에서 많은 지점(구독자)으로 전송되며, 이 구간에서 구독자에서 헤드엔드로 신호를 되돌려 보낼 필요가 없음을 의미합니다. 따라서 하류는 여러 신호가 간섭할 수 있는 양방향 또는 다점간 통신과 관련된 경쟁 또는 충돌 문제가 일반적으로 없습니다.
	</Answer>

9. HFC, DSL, and FTTH are all used for residential access. For each of these access technologies, provide a range of transmission rates and comment on whether the transmission rate is shared or dedicated.<br/>
	HFC, DSL 및 FTTH는 모두 가정용 접근에 사용됩니다. 각 접근 기술에 대해 전송 속도 범위를 제공하고 전송 속도가 공유되는지 전용인지 설명하세요.
	<Answer>
		**HFC**: HFC는 다양한 전송 속도를 제공합니다. 일반적으로 HFC 네트워크는 25Mbps에서 1Gbps 사이의 속도를 제공합니다. HFC 네트워크는 사용자 간에 전송 용량을 공유하므로 사용자가 동시에 사용할 때 속도가 변동할 수 있습니다.<br/><br/>
		**DSL**: DSL은 다양한 전송 속도를 제공합니다. 일반적으로 DSL은 1Mbps에서 100Mbps 사이의 속도를 제공합니다. DSL은 전통적인 전화선을 통해 데이터를 전송하므로 전송 용량이 사용자 간에 공유됩니다.<br/><br/>
		**FTTH**: FTTH는 다양한 전송 속도를 제공합니다. 일반적으로 FTTH는 100Mbps에서 1Gbps 이상의 속도를 제공합니다. FTTH는 광섬유를 사용하여 데이터를 전송하므로 전송 용량이 사용자 간에 공유되지 않습니다.
	</Answer>

### Section 1.3 network core - packet switching, circuit switching, network structure
11. Suppose there is exactly one packet switch between a sending host and a receiving host. The transmission rates between the sending host and the switch and between the switch and the receiving host are R1 and R2, respectively. Assuming that the switch uses store-and-forward packet switching, what is the total end-to-end delay to send a packet of length L? (Ignore queuing, propagation delay, and processing delay.)<br/>
	송신 호스트와 수신 호스트 사이에 정확히 하나의 패킷 스위치가 있다고 가정합니다. 송신 호스트와 스위치 간, 스위치와 수신 호스트 간의 전송 속도는 각각 R1 및 R2입니다. 스위치가 저장 및 전달 패킷 스위칭을 사용한다고 가정할 때, 길이 L의 패킷을 보내는 데 필요한 총 종단 간 지연 시간은 얼마인가요? (대기, 전파 지연 및 처리 지연을 무시하세요.)
	<Answer>
		**End-to-end delay = (L/R1) + (L/R2)**
	</Answer>

12. What advantage does a circuit-switched network have over a packet-switched network? What advantages does TDM have over FDM in a circuit-switched network?<br/>
	회선 교환 네트워크가 패킷 교환 네트워크에 비해 어떤 이점이 있나요? 회선 교환 네트워크에서 TDM이 FDM에 비해 어떤 이점이 있나요?
	<Answer>
		**회선 교환 네트워크**: 통신 회선을 설정하여 데이터를 교환하는 방식(전용선)<br/>
		**패킷 교환 네트워크**: 데이터를 패킷 단위로 전송하는 방식(여러 경로 이용)

		**회선 교환 네트워크의 장점**은 전송 중에 연결이 유지되므로 전송 중에 데이터가 손실되지 않는다는 것입니다. 이는 실시간 음성 통화 또는 비디오 스트리밍과 같은 실시간 응용 프로그램에 적합합니다. 반면 **패킷 교환 네트워크**는 데이터를 패킷 단위로 전송하며, 패킷이 도착하는 순서가 보장되지 않을 수 있습니다. 이는 데이터 전송 중에 패킷 손실이 발생할 수 있다는 것을 의미합니다.<br/><br/>
		**회선 교환 네트워크**에서 **TDM**은 **FDM**에 비해 **더 많은 사용자를 지원**할 수 있습니다. TDM은 시간을 여러 사용자 사이에 분할하여 전송하므로 여러 사용자가 동시에 전송할 수 있습니다. 반면 FDM은 주파수를 여러 사용자 사이에 분할하여 전송하므로 동시에 전송할 수 있는 사용자 수가 제한됩니다.
	</Answer>

13. Suppose users share a 2 Mbps link. Also suppose each user transmits continuously at 1 Mbps when transmitting, but each user transmits only 20 percent of the time. (See the discussion of statistical multiplexing in Section 1.3.)<br/>
	a. When circuit switching is used, how many users can be supported?<br/>
	b. For the remainder of this problem, suppose packet switching is used. Why will there be essentially no queuing delay before the link if two or fewer users transmit at the same time? Why will there be a queuing delay if three users transmit at the same time?<br/>
	c. Find the probability that a given user is transmitting. <br/>
	d. Suppose now there are three users. Find the probability that at any given time, all three users are transmitting simultaneously. Find the fraction of time during which the queue grows.<br/><br/>
	사용자가 2 Mbps 링크를 공유한다고 가정합니다. 또한 각 사용자가 전송할 때 1 Mbps로 연속적으로 전송하지만 각 사용자가 전송하는 시간은 20%뿐이라고 가정합니다. (1.3절의 통계적 다중화에 대한 토론을 참조하세요.)<br/>
	a. 회선 교환을 사용할 때 지원할 수 있는 사용자 수는 몇 명입니까?<br/>
	b. 이 문제의 나머지 부분에서 패킷 교환을 사용한다고 가정합니다. 두 명 이하의 사용자가 동시에 전송할 때 링크 앞에 거의 대기 지연이 없는 이유는 무엇인가요? 세 명의 사용자가 동시에 전송할 때 대기 지연이 발생하는 이유는 무엇인가요?<br/>
	c. 주어진 사용자가 전송 중인 확률을 찾으세요.<br/>
	d. 이제 세 명의 사용자가 있다고 가정합니다. 임의의 시점에서 세 명의 사용자가 동시에 전송 중인 확률을 찾으세요. 대기열이 증가하는 시간의 분수를 찾으세요.
	<Answer>
		a. **회선 교환을 사용할 때 지원할 수 있는 사용자 수는 2명입니다.** 각 사용자가 전송할 때 1 Mbps로 연속적으로 전송하지만 각 사용자가 전송하는 시간은 20%뿐이므로 2명의 사용자가 전송할 때 2 Mbps 링크를 공유할 수 있습니다.<br/><br/>
		b. **패킷 교환을 사용할 때**, 두 명 이하의 사용자가 동시에 전송할 때 링크 앞에 거의 대기 지연이 없는 이유는 **통계적 다중화**입니다. 두 명 이하의 사용자가 동시에 전송할 때 링크 앞에 대기열이 거의 없는 이유는 각 사용자가 전송하는 시간이 20%뿐이기 때문에 링크가 거의 사용되지 않기 때문입니다. 세 명의 사용자가 동시에 전송할 때 대기 지연이 발생하는 이유는 링크가 사용 중이므로 패킷이 대기열에 추가되어야 하기 때문입니다.<br/><br/>
		c. **주어진 사용자가 전송 중인 확률은 20%입니다.** 각 사용자가 전송하는 시간이 20%이므로 각 시간 단위에 사용자가 전송 중인 확률은 20%입니다.<br/><br/>
		d. **세 명의 사용자가 동시에 전송 중인 확률은 0.008입니다.** 각 사용자가 전송하는 시간이 20%이므로 세 명의 사용자가 동시에 전송 중인 확률은 0.2^3 = 0.008입니다. 대기열이 증가하는 시간의 분수는 0.2입니다.

		**네트워크 계층(Network Layer)**: 데이터를 목적지로 전달하기 위해 경로를 선택하고 주소를 지정합니다.<br/>
		**전송 계층(Transport Layer)**: 데이터 전송 속도를 조절하고 흐름 제어를 수행합니다.<br/>
		**응용 계층(Application Layer)**: 사용자와 네트워크 간의 상호 작용을 지원하고 데이터를 처리합니다.<br/><br/>
		**이러한 각 계층의 주요 책임**은 다음과 같습니다:<br/><br/>
		**물리 계층**: 데이터를 전송하기 위해 전송 매체를 사용합니다.<br/>
		**데이터 링크 계층**: 물리적 매체를 통해 데이터를 전송하고, 오류 감지 및 수정을 수행합니다.<br/>
		**네트워크 계층**: 데이터를 목적지로 전달하기 위해 경로를 선택하고 주소를 지정합니다.<br/>
		**전송 계층**: 데이터 전송 속도를 조절하고 흐름 제어를 수행합니다.<br/>
		**응용 계층**: 사용자와 네트워크 간의 상호 작용을 지원하고 데이터를 처리합니다.
	</Answer>

24. What is an application-layer message? A transport-layer segment? A network-layer datagram? A link-layer frame?<br/>
	응용 계층 메시지는 무엇인가요? 전송 계층 세그먼트는 무엇인가요? 네트워크 계층 데이터그램은 무엇인가요? 링크 계층 프레임은 무엇인가요?
	<Answer>
		**응용 계층 메시지**는 **응용 계층에서 생성된 데이터**입니다. 예를 들어, 이메일, 웹 페이지 요청 또는 파일 전송과 같은 데이터가 응용 계층 메시지가 될 수 있습니다.<br/><br/>
		**전송 계층 세그먼트**는 **전송 계층에서 생성된 데이터**입니다. 전송 계층 세그먼트에는 송신 및 수신 호스트의 포트 번호, 시퀀스 번호, 확인 응답 번호 등의 정보가 포함될 수 있습니다.<br/><br/>
		**네트워크 계층 데이터그램**은 **네트워크 계층에서 생성된 데이터**입니다. 네트워크 계층 데이터그램에는 출발지 및 목적지 주소, 프로토콜 정보 등의 정보가 포함될 수 있습니다.<br/><br/>
		**링크 계층 프레임**은 **링크 계층에서 생성된 데이터**입니다. 링크 계층 프레임에는 출발지 및 목적지 MAC 주소, 오류 감지 및 수정 정보 등의 정보가 포함될 수 있습니다.
	</Answer>

25. Which layers in the Internet protocol stack does a router process? Which layers does a link-layer switch process? Which layers does a host process?<br/>
	인터넷 프로토콜 스택의 어떤 계층을 라우터가 처리하나요? 어떤 계층을 링크 계층 스위치가 처리하나요? 어떤 계층을 호스트가 처리하나요?
	<Answer>
		**라우터**는 **네트워크 계층**과 **링크 계층**을 처리합니다. 라우터는 **네트워크 계층에서 패킷을 전달하고, 링크 계층에서 프레임을 전달**합니다.<br/><br/>
		**링크 계층 스위치**는 **링크 계층**을 처리합니다. 링크 계층 스위치는 **링크 계층에서 프레임을 전달**합니다.<br/><br/>
		**호스트**는 **응용 계층**, **전송 계층**, **네트워크 계층**, **링크 계층**을 처리합니다. 호스트는 **응용 계층에서 데이터를 생성하고, 전송 계층에서 세그먼트를 생성하고, 네트워크 계층에서 데이터그램을 생성하며, 링크 계층에서 프레임을 생성**합니다.
	</Answer>
### Section 1.6 Security
26. What is self-replicating malware?<br/>
	자체 복제 악성 코드는 무엇인가요?
	<Answer>
		**자체 복제 악성 코드**는 **자동으로 복제되는 악성 코드**입니다. 자체 복제 악성 코드는 **감염된 시스템에서 자동으로 복제**되어 다른 시스템으로 전파될 수 있습니다. 이러한 악성 코드는 **사용자의 동의 없이 시스템에 침투**할 수 있으며, **다른 시스템에 전파**될 수 있습니다.
	</Answer>

27. Describe how a botnet can be created and how it can be used for a DDoS attack.<br/>
	봇넷을 생성하는 방법과 DDoS 공격에 사용하는 방법을 설명하세요.
	<Answer>
		**봇넷**은 **여러 컴퓨터에 악성 코드를 설치하여 제어하는 네트워크**입니다. 봇넷은 **악성 코드를 설치한 컴퓨터를 원격으로 제어**할 수 있으며, **대규모 DDoS(분산 서비스 거부) 공격**에 사용될 수 있습니다.<br/><br/>
		**봇넷을 생성하는 방법**은 다음과 같습니다:<br/><br/>
		**악성 코드 설치**: 악성 코드를 설치하여 컴퓨터를 제어합니다.<br/>
		**제어 서버 연결**: 제어 서버에 연결하여 봇넷을 제어합니다.<br/>
		**봇넷 제어**: 봇넷을 사용하여 다양한 공격을 수행합니다.<br/><br/>
		**DDoS 공격에 사용하는 방법**은 다음과 같습니다:<br/><br/>
		**봇넷 제어**: 봇넷을 사용하여 대규모 DDoS 공격을 수행합니다.<br/>
		**대상 공격**: DDoS 공격 대상에 대량의 트래픽을 보냅니다.<br/>
		**서비스 거부**: 대상 서버가 과부하되어 서비스를 거부하게 만듭니다.
	</Answer>
