## 정보보안과 암호학 개요
### 컴퓨터 보안
컴퓨터 보안이란?
> 자동화된 정보 시스템이 정보의 무결성, 가용성, 기밀성을 보존하는 목적을 달성하기 위해 제공되는 보호기능.
보호대상: 하드웨어, 소프트웨어, 펌웨어, 정보/데이터 및 통신망

- CIA triad
	- Confidentiality(기밀성)
	- Integrity(무결성)
	- Availability(가용성)
	- **추가 요구사항**: Authenticity(인증성), Accountability(책임성)
### 취약점, 공격 및 대책 유형
- 취약점
	- 손상됨(무결성 손실)
	- 누설됨(기밀성 손실)
	- 사용할 수 없거나 매우 느림(가용성 손실)
- 공격
	- 수동적 공격 vs 능동적 공격
	- 내부 공격 vs 외부 공격
- 대책
	- 예방, 탐지, 복구
### 공격 표면 (Attack Surface)
> 공격자가 공격을 시도할 수 있는 모든 경로

네트워크, 소프트웨어, 인간 등이 있음
### 컴퓨터 보안 전략(Computer Security Strategy)
- 보안 정책(Security policy): 시스템이나 조직이 민감하고 중요한 시스템 자원을 보호하기 위한 서비스를 제공하는 방법을 규정하는 규칙과 관행
- 보안 구현(Security implementation): 예방, 탐지, 대응, 복구 등의 과정
- 보안 확신(Security assurance): 보안 조치가 의도한 대로 시스템과 그 정보를 보호하는 데 얼마나 효과적인지에 대한 신뢰도
- 보안 평가(Security evaluation): 특정 기준에 대해 컴퓨터 제품이나 시스템을 검사하는 과정

## 암호학 개요 (Introduction to Cryptography)
### 암호화 (Encryption)
> 데이터 기밀성을 제공하는 기술,
평문(plain text)을 암호문(cipher text)으로 변환하는 과정

- 대칭키 암호화(Symmetric-key cryptography)
	- 블록 암호
	- 스트림 암호
- 공개키 암호화(Public-key cryptography)
	- 인수분해 기반
	- 이산 로그 기반
	- 포스트-퀀텀 완전 암호화: 격자 기반, 코드 기반 등

다음 세 가지 알고리즘으로 구성된다.
- 설정 알고리즘 : $Setup(\lambda ) → {K_{enc}, K_{dec}}$
- 암호화 알고리즘: $Enc({K_{enc}, M}) → CT$
- 복호화 알고리즘: $Dec({K_{dec}, CT}) → M$

### 대칭 암호화 (Symmetric Encryption)
> 암호화와 복호화에 같은 키를 사용하는 암호화 방식

개인 키 암호화라고도 부름
전통적인 암호화 방식으로 블록 암호화(DES, AES, ARIA, SEED), 스트림 암호화(RC4, ChaCha) 등이 있음
장점: 비대칭 암호화 방식에 비해 빠르다
단점: 키 공유 문제, 키의 개수가 많다 -> $N(N-1)/2$개의 키 가짐

### 비대칭 암호화 (Asymmetric Encryption)
> 암호화와 복호화에 서로 다른 키를 사용하는 암호화 방식

공개 키 암호화라고도 부름
RSA, ElGamal, NTRU 등이 있음
장점: 키 공유 문제 해결, 키 개수가 적다 -> $2N$개의 키 가짐
단점: 대칭 암호화 방식에 비해 느리다

### 암호화 관련 용어 (Terminology for Encryption)
- 평문(plain text): 알고리즘에 입력으로 제공되는 원래의 메시지나 데이터
- 암호문(cipher text): 암호화 알고리즘의 출력물
- 암호화 알고리즘(encryption algorithm): 평문을 암호문으로 변환하는 알고리즘
- 복호화 알고리즘(decryption algorithm): 암호문을 평문으로 변환하는 알고리즘
- 공개키와 비밀키(public key and private key): 각각 암호화와 복호화에 선택된 키 쌍 

### 케르호프의 원리 (Kerckhoffs's Principle)
> 암호화 알고리즘은 공개되어도 안전해야 한다. 비밀키만 안전하면 된다.

시스템의 안정성을 높이고, 다양한 키를 사용하여 다른 사람들과 소통하는 것을 용이하게 한다.

### 암호화 공격의 종류 (Basic Types of Attacking against Encryption)
1. 브루트 포스 공격(Brute-force attack): 모든 가능한 키를 시도하여 암호문을 해독하는 공격
2. 암호 분석(Cryptanalysis): 암호화 알고리즘의 구조를 분석하여 평문을 추정하는 공격
	- 암호문 단독 공격(ciphertext-only attack)
	- 알려진 평문 공격(known-plaintext attack)
	- 선택 평문 공격(chosen-plaintext attack)
	- 선택 암호문 공격(chosen-ciphertext attack)

## 일반적인 암호화 기법 (Classical Encryption)
### 표기법 정리 (Notation)
```math
Z_m = {0, 1, ..., m - 1}\\
g mod p: g를 p로 나눈 나머지\\
10 mod 3 = 1\\
729 mod 31 = 16 (729 = 23 * 31 + 16)\\
-7 mod 26 = 19 (-7 = 26 * (-1) + 19)\\
평문 공간 = 암호문 공간 = 알파벳 문자들 = \mathbb{Z}_{26}
```

### 시프트 암호 (Shift Cipher)
평문의 각 문자를 고정된 수만큼 이동시키는 암호화 방식
```math
0 <= k <= 25\\
Enc(K, x) = (x + K) mod \;26 \;\;\;(cf. K = 3: 시저 암호)\\
Dec(K, y) = (y - K) mod \;26\\
```
예시: K = 9
암호화
shift (18 7 8 5 19) → +9 mod 26 → (1 16 17 14 2) BQROC
복호화
BQROC (1 16 17 14 2) → -9 mod 26 → (18 7 8 5 19) shift

공격
무차별 대입 공격: 비밀키에 대한 후보가 26개뿐이다.
알려진 평문 공격: 예시의 (shift, BQROC)의 경우, K = B - s = (1 - 18) mod 26 = 9

### 아핀 암호 (Affine Cipher)
선형 대수학과 모듈러 연산을 이용한 암호화 방식
```math
For K = (\alpha ,\beta ) where \alpha, \beta \in \mathbb{Z}_{26} and gcd(\alpha, 26) = 1\\
Enc(K, x) = (\alpha x + \beta) mod \;26\\
Dec(K, y) = \alpha^{-1} (y - \beta) mod \;26\\
```

공격 방식

- 알려진 평문 공격: 예시의 (hot, AXG)의 경우 -> 연립방정식
	- $7\alpha + \beta = 0\,mod\,6$
	- $14\alpha + \beta = 23\,mod\,6$

### 치환 암호 (Substitution Cipher)
평문의 각 문자를 다른 문자로 대체하는 암호화 방식
```math
Enc(K, x) = \pi(x)\\
Dec(K, y) = \pi^{-1}(y)\\
```
예시
```math
\pi = \{A(0) \rightarrow 2, B(1) \rightarrow 7, C(2) \rightarrow 9, ..., Z(25) \rightarrow 19\}\\
Enc(\pi, C) = \pi(C) = 9 = J \\
Dec(\pi, J) = \pi^{-1}(J) = C
```

공격 방식
- 브루트 포스 공격: 26!개의 키 후보 (약 4 * 10^26)
- 평문 선택 공격: n개의 평문-암호문 쌍을 이용하여 키를 찾는 방식
