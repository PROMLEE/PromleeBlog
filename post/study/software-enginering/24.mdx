## 정적 vs 동적 객체 설계 (Static vs Dynamic Object Design)
**동적 모델**: 코드 또는 메서드 본문의 로직과 동작 설계
- 예: UML 상호작용 다이어그램, 시퀀스 다이어그램, 커뮤니케이션 다이어그램
- 더 흥미롭고, 어렵고, 중요한 다이어그램이 되는 경향이 있음

**정적 모델**: 패키지, 클래스, 속성 및 메서드 서명 설계(메서드 본문 제외)
- 예: UML 클래스 다이어그램, 패키지 다이어그램, 객체 다이어그램

*애자일 모델링 실천*
- 상호작용 다이어그램(동적)에 짧은 시간을 투자한 후 관련 클래스 다이어그램(정적)을 작성

## 객체 설계 기술의 중요성 (Importance of Object Design Techniques)
*UML* 작성은 설계에 대한 결정을 반영하는 것이 아니라 설계 결정을 돕는 것이다.
중요한 것은 객체 설계 기술이지 UML을 그리는 방법이 아니다.
**기본적인 객체 설계 요구 지식**
- 책임 할당 원칙(Responsibility Assignment Principle)
- 디자인 패턴(Design Pattern)
